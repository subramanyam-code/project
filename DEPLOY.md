# TSPM — Deployment Guide

## Architecture

```
┌─────────────────────┐        ┌──────────────────────┐
│   Netlify (Free)    │        │   Render.com (Free)  │
│                     │        │                      │
│  Next.js 14 Frontend│──/api/*│  FastAPI Backend     │
│  (SSR via plugin)   │───────▶│  + PostgreSQL DB     │
└─────────────────────┘        └──────────────────────┘
```

## Local Development (Docker)

Everything runs inside the project folder:

```
project/
└── database/
    ├── postgres_data/   ← PostgreSQL data files
    ├── redis_data/      ← Redis data files
    └── pgadmin_data/    ← pgAdmin settings
```

```bash
# Start all services
docker compose up -d

# View logs
docker compose logs -f

# Stop everything
docker compose down
```

| Service  | URL                        | Credentials             |
|----------|----------------------------|-------------------------|
| Frontend | http://localhost:3000      | —                       |
| Backend  | http://localhost:8000      | —                       |
| API Docs | http://localhost:8000/api/v1/docs | —               |
| pgAdmin  | http://localhost:5050      | admin@admin.com / admin |

### Connect pgAdmin to the local database
1. Open http://localhost:5050
2. Add New Server:
   - Name: `tspm_local`
   - Host: `db`
   - Port: `5432`
   - Database: `hidb`
   - Username: `mydata`
   - Password: `mypassword`

## Production Deployment

### Step 1 — Deploy Backend to Render.com

1. Push this repo to GitHub
2. Go to https://render.com → New → Blueprint
3. Connect your GitHub repo — Render detects `render.yaml` automatically
4. It will create:
   - `tspm-backend` web service (FastAPI via Docker)
   - `tspm-db` managed PostgreSQL database
   - Alembic migrations run automatically on every deploy
5. After deploy, copy your backend URL:
   ```
   https://tspm-backend.onrender.com
   ```

### Step 2 — Deploy Frontend to Netlify

1. Go to https://netlify.com → Add new site → Import from Git
2. Connect your GitHub repo
3. Netlify auto-detects `netlify.toml` — settings are pre-configured:
   - Base directory: `frontend`
   - Build command: `npm run build`
   - Publish directory: `.next`
4. Add environment variable in Netlify dashboard:
   - `NEXT_PUBLIC_API_URL` = (leave blank — frontend uses `/api/*` proxy)
   - `NEXT_PUBLIC_APP_NAME` = `TeamStatus`
5. After deploy, copy your Netlify URL:
   ```
   https://your-site.netlify.app
   ```

### Step 3 — Connect Frontend ↔ Backend

After both are deployed, update two places:

**In `netlify.toml`** (then redeploy frontend):
```toml
[[redirects]]
  from = "/api/*"
  to   = "https://tspm-backend.onrender.com/api/:splat"   # ← your Render URL
```

**In `render.yaml`** (then redeploy backend):
```yaml
- key: FRONTEND_URL
  value: https://your-site.netlify.app    # ← your Netlify URL

- key: BACKEND_CORS_ORIGINS
  value: https://your-site.netlify.app    # ← same
```

Or set these directly in the Render dashboard under Environment Variables — no redeploy needed.

### Step 4 — Set Secret Environment Variables on Render

In Render dashboard → tspm-backend → Environment, add:
- `AWS_ACCESS_KEY_ID`
- `AWS_SECRET_ACCESS_KEY`
- `AWS_S3_BUCKET`
- `SMTP_HOST`, `SMTP_USER`, `SMTP_PASSWORD`, `FROM_EMAIL`

`SECRET_KEY` is auto-generated by Render (`generateValue: true` in render.yaml).

## Keeping Data in the Project Folder (Local Only)

The `database/` folder stores all data locally — it is gitignored:
```
database/postgres_data/    ← PostgreSQL binary files
database/redis_data/       ← Redis dump.rdb
database/pgadmin_data/     ← pgAdmin config/sessions
```

> **Note:** Production database on Render is separate — it's a managed cloud PostgreSQL
> instance, not the local folder. The local folder is only for your dev machine.
